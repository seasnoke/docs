The worker is a Rust process that waits for run tasks, executes an agent loop in an ephemeral microVM sandbox, and publishes live telemetry back to Redis.

## Queue behavior

The worker blocks on:

```rust
let (_, task_json): (String, String) = redis::cmd("BRPOP")
    .arg("seasnoke:tasks:pending")
    .arg(0)
    .query_async(&mut conn)
    .await?;
```

Each task contains:

| Field | Description |
| --- | --- |
| `id` | Task UUID. |
| `run_id` | Parent run UUID. |
| `repo_url` | Repository URL supplied by the client. |
| `prompt` | User task prompt. |
| `task_type` | Current API values are `microvm-1`, `microvm-2`, and `microvm-3`. |

## Sandbox lifecycle

For each task, the worker creates a microsandbox named `seasnoke-{task_id}` with:

| Setting | Value |
| --- | --- |
| Image | `alpine` |
| CPUs | `1` |
| Memory | `512` MB |

If sandbox creation fails, the worker publishes diagnostic messages for KVM, image pull, and resource errors, then emits `TASK_STATUS:{task_id}:FAILED`.

## Agent loop

The agent loop calls Anthropic Messages API with model `claude-3-5-sonnet-20241022`. The system prompt requires the model to emit XML actions:

| Tag | Meaning |
| --- | --- |
| `<execute>...</execute>` | Run a shell command in the sandbox and stream stdout/stderr. |
| `<finish>...</finish>` | Mark the task complete and extract the staged git diff. |

The loop allows up to 15 iterations. On success, it stages sandbox changes, captures `git diff --staged`, base64-encodes it, and publishes:

```rust
let result_msg = format!("TASK_RESULT:{}:SUCCEEDED:{}", task_id, diff_b64);
conn.publish::<_, _, ()>(&log_channel, result_msg).await?;
```

The worker then stops the sandbox and pushes the result JSON to `seasnoke:tasks:completed`.
