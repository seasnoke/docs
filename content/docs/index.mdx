SeaSnoke is an agent-native software development platform for running coding agents in parallel and inspecting their work as it happens.

The current project is split across four local repositories:

| Repository | Role | Main stack |
| --- | --- | --- |
| `web` | Product website, dashboard, docs shell, settings, graph, and run detail UI. | TanStack Start, React 19, Chakra UI, React Flow |
| `api` | Backend run creation and Server-Sent Events stream endpoint. | Rust, Actix Web, SQLx, PostgreSQL, Redis |
| `worker` | Background worker that consumes queued tasks and runs agents in microVM sandboxes. | Rust, Redis, microsandbox, Anthropic Messages API |
| `app` | Lightweight run console UI that creates runs, subscribes to SSE, and displays diffs/logs. | Vite, React, Zustand |

## How a run works

1. A client submits a repository URL and prompt to the API.
2. The API creates a run row in PostgreSQL and enqueues three Redis task messages.
3. A worker blocks on `seasnoke:tasks:pending`, creates an Alpine microsandbox for each task, and runs the agent loop.
4. The worker publishes logs, task status, and base64-encoded diffs to `seasnoke:logs:{run_id}`.
5. Clients subscribe to `/api/runs/{id}/stream` and update the UI from Server-Sent Events.

## Local services

The API reads these defaults:

```rust
let database_url = std::env::var("DATABASE_URL")
    .unwrap_or_else(|_| "postgres://seasnoke:seasnoke@127.0.0.1:5432/seasnoke".into());
let redis_url = std::env::var("REDIS_URL")
    .unwrap_or_else(|_| "redis://127.0.0.1:6379".into());
let host = std::env::var("HOST").unwrap_or_else(|_| "127.0.0.1".into());
let port = std::env::var("PORT").unwrap_or_else(|_| "8080".into());
```

The worker also uses `REDIS_URL` and requires `ANTHROPIC_API_KEY`.

## Current scope

The product UI has polished demo surfaces and mocked dashboard data. The API and worker contain the live run pipeline: run creation, task queueing, Redis Pub/Sub streaming, sandbox execution, Anthropic calls, and diff extraction.
