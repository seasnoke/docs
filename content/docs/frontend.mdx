SeaSnoke currently has two frontend-oriented repositories.

## Web

The `web` repository is the polished product UI and demo shell. It uses TanStack Start with React 19, Chakra UI v3, Framer Motion, and React Flow.

Main routes:

| Route | Purpose |
| --- | --- |
| `/` | Landing page. |
| `/dashboard` | Dashboard overview with run cards and stats. |
| `/tasks/new` | New agent task form. |
| `/runs/$id` | Run detail workspace. |
| `/graph` | Visual change graph. |
| `/settings` | Integrations and workspace settings. |
| `/docs` | Existing in-app documentation shell. |

The `web` README notes that dashboard data is mocked in `src/lib/mock-data.ts`; it is built as a realistic SaaS demo surface rather than a fully wired production frontend.

## App

The `app` repository is a smaller run console. It keeps run state in Zustand and subscribes to the API stream with `EventSource`.

Its SSE hook defaults to:

```ts
const API_BASE = import.meta.env.VITE_API_URL || 'http://127.0.0.1:8080';
```

The client parses these messages:

| Message | Behavior |
| --- | --- |
| `TASK_RESULT:{task_id}:SUCCEEDED:{base64_diff}` | Decode the diff, mark the task successful, and attach the diff to the task. |
| `TASK_STATUS:{task_id}:FAILED` | Mark the task failed. |
| Other log lines | Append to the terminal log. |

This makes `app` useful for watching the live backend/worker pipeline while `web` provides the broader product experience.
